Writing Helper Functions 2

In this example, we've defined another helper function isNotNullOrBlank() by negating the value return from standard library function isNullOrBlank().

While isNullOrBlank() may helper the compiler infer nullability (thanks to Contracts), the compiler cannot extend those inferences to a function calling into another function that includes a contract.

This highlights the need for helper functions to define their own contracts - rather than relying on the contracts of functions they may use in their implementation.

Improving Compiler Inferences

In the next examples, we'll improve these helper functions by making use of Kotlin's Contract apis.
